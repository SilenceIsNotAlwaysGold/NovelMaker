# 一致性保证机制

> 长篇小说最容易翻车的地方不是文笔不好，而是前后矛盾。

## 一、写前必读流程

每次开始写新章节前，必须完成以下检查：

### 1. 分层记忆加载
```
✅ 读取全书基因（核心设定不能忘）
✅ 读取卷概要（本卷目标和角色状态）
✅ 读取当前弧线（当前小故事在讲什么）
✅ 读取最近3章摘要（精确衔接）
✅ 读取上一章最后500字（语气/场景/悬念衔接）
```

### 2. 伏笔检索
- 本章是否有计划回收的伏笔（查伏笔追踪表）
- 上一章是否埋了需要本章呼应的伏笔

### 3. 角色状态确认
- 本章出场角色在哪里？上一章结束时做了什么？
- 角色当前的情绪状态是什么？

## 二、六大一致性维度

### 维度1：人设一致性

| 检查项 | 方法 | 常见错误 |
|--------|------|----------|
| 性格 | 角色行为是否符合已建立的性格 | 内向角色突然变话痨 |
| 口头禅 | 是否保持角色的标志性用语 | 忘了角色的口头禅 |
| 习惯动作 | 紧张/高兴时的习惯动作是否一致 | 紧张咬嘴唇的角色改成搓手 |
| 知识边界 | 角色知道的信息是否超出了ta应该知道的 | 角色A知道了只有B知道的秘密 |
| 能力边界 | 角色能力是否突然暴涨或退化 | 上章还打不过的人这章轻松碾压 |

**自查方法**：写完一个角色的对话/行为后，问自己"把角色名遮住，读者能不能猜出是谁？"如果不能，说明角色特征不够鲜明。

### 维度2：时间线一致性

| 检查项 | 常见错误 |
|--------|----------|
| 日/夜交替 | 明明是晚上，下一段突然出太阳了 |
| 事件耗时 | 走路去某地，上一章说要三天，结果一章就到了 |
| 年龄变化 | 时间跨度一年，角色年龄没变 |
| 季节变化 | 冬天过去了，但场景描写还是冬天 |
| 并行事件 | A做某事时B在做另一件事，时间线对不上 |

**管理方法**：在卷概要中维护一个简易时间轴：
```
第1章：第一天 白天 - 林远到达云隐城
第2章：第一天 夜晚 - 林远发现纹身异变
第3章：第二天 早上 - 入门考核
第4章：第二天 下午 - 遇见苏晴
...
```

### 维度3：空间一致性

| 检查项 | 常见错误 |
|--------|----------|
| 角色位置 | 角色在A城，但对话提到"刚从A城出发" |
| 场景布局 | 房间的门上章在东，这章在西 |
| 距离逻辑 | 两个城市忽近忽远 |
| 同场人物 | 场景中在场的人突然消失又出现 |

**管理方法**：每章开头确认"谁在哪里"，结尾确认"谁去了哪里"。

### 维度4：设定一致性

| 检查项 | 常见错误 |
|--------|----------|
| 力量体系 | 修炼规则前后矛盾 |
| 社会规则 | 世界观设定被无视 |
| 道具/功法 | 同一功法效果描述不同 |
| 称谓体系 | 称呼方式混乱 |

**管理方法**：所有专有名词必须在术语表中登记。有疑问时查术语表，不要凭记忆。

### 维度5：情感一致性

| 检查项 | 常见错误 |
|--------|----------|
| 情绪连贯 | 上章悲痛欲绝，下章若无其事 |
| 关系发展 | 还没和好就恢复亲密 |
| 动机连续 | 角色的目标突然变了，没有铺垫 |
| 创伤影响 | 经历重大事件后毫无影响 |

**关键规则**：重大情感事件至少影响后续2-3章的角色行为。不能只影响一章就恢复正常。

### 维度6：叙事一致性

| 检查项 | 常见错误 |
|--------|----------|
| 叙述视角 | 第三人称突然混入第一人称 |
| 信息泄露 | 作者视角的信息泄露给受限视角的角色 |
| 语言风格 | 古代背景出现现代网络用语 |
| 叙述密度 | 前面章节事无巨细，后面突然变成流水账 |

## 三、写后一致性检查清单

每章写完后逐项检查：

### 人设检查
- [ ] 每个角色的行为是否符合已建立的性格
- [ ] 角色的说话风格是否保持一致
- [ ] 角色知道的信息没有超出其应知范围
- [ ] 角色的能力没有无故暴涨或退化

### 时空检查
- [ ] 时间线连贯（日夜、天数、季节）
- [ ] 角色位置合理（没有瞬移）
- [ ] 场景描写与之前一致（房间布局、地理位置）

### 设定检查
- [ ] 力量体系/世界规则没有被违反
- [ ] 专有名词使用与术语表一致
- [ ] 道具/功法效果描述与之前一致

### 情感检查
- [ ] 角色情绪从上一章自然过渡
- [ ] 重大事件的情感影响在持续
- [ ] 关系发展有足够铺垫

### 叙事检查
- [ ] 叙述视角保持统一
- [ ] 没有作者视角信息泄露
- [ ] 前文提到的悬念至少有呼应
- [ ] 没有遗忘场景中在场的角色

## 四、跨章一致性管理

### 衔接句检查
上下章衔接时，注意：
```
上章结尾：林远握紧了剑柄，向黑暗中走去。
下章开头：✅ 黑暗中传来金属碰撞的声音。林远停下脚步。
下章开头：❌ 第二天清晨，林远在客栈醒来。（时间跳跃无交代）
```

### 信息追踪
当角色获得重要信息时，标记：
- **谁知道了什么**：角色A在第X章得知秘密Y
- **谁还不知道**：角色B在第X章时仍不知道秘密Y
- **何时可以揭示**：角色B在第Z章发现真相

### 状态传递
每章摘要中必须记录影响后续章节的状态变化：
- 角色受伤 → 后续章节行动受限
- 角色获得道具 → 后续章节可以使用
- 角色关系变化 → 后续对话语气调整
- 环境变化 → 后续场景描写更新
