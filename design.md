# 中文长篇小说写作智能体 - 架构设计 v2

> 目标：支撑 20万-100万字 长篇网文创作，解决超长篇幅下的记忆丢失、角色漂移、剧情失控问题

## 一、为什么 v1 设计撑不住几十万字

| 问题 | 原因 | 影响 |
|------|------|------|
| 摘要链膨胀 | 60章×500字=3万字，吃光上下文 | 后期章节质量断崖式下降 |
| 人物档案是静态的 | 主角第1章和第50章应该完全不同 | 角色没有成长弧线 |
| 大纲太扁平 | 线性TODO管不了多线叙事 | 支线剧情失控 |
| 伏笔无人管 | 没有机制追踪"埋下→回收" | 挖坑不填，读者骂娘 |
| 没有分卷概念 | 60章铁板一块 | 节奏无法分段控制 |

## 二、核心设计理念：分层记忆 + 分卷管控

### 关键洞察

人类作者写长篇时，脑子里并不是记住每一个字。而是：
- **模糊记忆**：大致知道前面发生了什么（全书摘要）
- **结构记忆**：知道当前在哪个阶段、哪条线（卷/弧线定位）
- **精确记忆**：清楚记得最近几章的细节（近期上下文）
- **索引记忆**：知道某个伏笔在哪里埋的，需要时翻回去看（伏笔追踪表）

智能体的记忆系统应该模仿这个结构，而不是把所有信息平铺塞进上下文。

## 三、分层记忆架构

```
┌─────────────────────────────────────────────────┐
│  Layer 0: 基因层（永久 · 每章必读 · <2000字）     │
│  ├── 核心设定卡：主角姓名/金手指/核心矛盾         │
│  ├── 世界规则摘要：力量体系/社会结构（压缩版）     │
│  └── 全书节奏图：当前在哪个阶段，整体走向          │
├─────────────────────────────────────────────────┤
│  Layer 1: 卷级记忆（当前卷 · 每章必读 · <3000字） │
│  ├── 当前卷大纲：本卷目标/冲突/高潮设计            │
│  ├── 本卷角色状态：经历了什么变化，当前心态         │
│  ├── 活跃支线清单：本卷在推进的剧情线              │
│  └── 本卷伏笔状态：已埋/待回收/已回收              │
├─────────────────────────────────────────────────┤
│  Layer 2: 弧线记忆（当前弧 · 每章必读 · <2000字） │
│  ├── 当前剧情弧概要：这段故事在讲什么              │
│  ├── 弧线内角色动态：谁在场、关系变化              │
│  └── 弧线节奏定位：起承转合走到哪一步了            │
├─────────────────────────────────────────────────┤
│  Layer 3: 章级记忆（最近3章 · 精确衔接 · <2000字）│
│  ├── 最近3章摘要（各300-500字）                    │
│  ├── 上一章最后500字原文                           │
│  └── 当前章大纲目标                               │
├─────────────────────────────────────────────────┤
│  Layer 4: 按需检索（不常驻 · 需要时才加载）        │
│  ├── 历史卷摘要（已完成的卷）                      │
│  ├── 完整人物档案（需要深入刻画某角色时）          │
│  ├── 伏笔详情（需要回收某个伏笔时）               │
│  └── 特定历史章节原文（需要呼应前文时）            │
└─────────────────────────────────────────────────┘

每章写作时的上下文预算：
Layer 0 (~2000字) + Layer 1 (~3000字) + Layer 2 (~2000字) + Layer 3 (~2000字) = ~9000字
留给正文生成的空间：充足
```

## 四、分卷叙事结构

### 4.1 层级关系

```
全书（20-100万字）
├── 第一卷：初入江湖（5-15万字）
│   ├── 弧线A：身世之谜（3-5章）
│   │   ├── 第01章
│   │   ├── 第02章
│   │   └── 第03章
│   ├── 弧线B：初遇强敌（3-5章）
│   │   ├── 第04章
│   │   └── ...
│   └── 弧线C：卷末高潮（2-3章）
├── 第二卷：风云变幻（5-15万字）
│   ├── 弧线D：...
│   └── ...
└── ...
```

### 4.2 每卷自治

每一卷是一个相对独立的"故事单元"，有自己的：
- **卷级目标**：这一卷主角要达成什么
- **卷级反派**：这一卷的主要对手
- **卷级成长**：主角在这一卷获得什么能力/认知
- **卷首悬念**：开卷就抛出的问题
- **卷末高潮**：本卷的最大冲突爆发点
- **卷间钩子**：读完本卷为什么想看下一卷

### 4.3 弧线（Arc）设计

每条弧线是一段连续的小故事（3-8章），有完整的起承转合：
- **起**：建立情境和悬念
- **承**：发展冲突和关系
- **转**：出现意外或反转
- **合**：小高潮或阶段性结局

弧线是**章节创作的直接上级管理者**，决定了每章的叙事目标和节奏定位。

## 六、伏笔追踪系统

这是长篇小说最容易翻车的地方。设计一个结构化的追踪表：

```markdown
# 伏笔追踪表

## 状态说明
- 🔴 已埋设：伏笔已埋下，等待回收
- 🟡 部分回收：已有暗示但未完全揭晓
- 🟢 已回收：伏笔已完整回收
- ⚫ 已废弃：决定不再回收（需注明原因）

## 伏笔清单

| ID | 伏笔描述 | 埋设位置 | 计划回收位置 | 状态 | 关联角色 |
|----|----------|----------|-------------|------|----------|
| F001 | 主角左手的神秘纹身 | 卷一·第3章 | 卷三·弧线G | 🔴 | 林远 |
| F002 | 师父临终前的那句话 | 卷一·第8章 | 卷二·弧线E | 🟡 | 林远、陈老 |
| F003 | 黑市商人的真实身份 | 卷一·第5章 | 卷一·第12章 | 🟢 | 赵九 |
```

**写前检查**：每章开写前，检索本章计划回收的伏笔，确保不遗漏。
**写后更新**：每章写完后，登记新埋的伏笔，更新回收状态。
**卷末审计**：每卷结束时，审计所有 🔴 状态伏笔，确认是否需要调整回收计划。

## 七、角色动态档案

不再用静态卡片，改为**版本化的角色状态**：

```markdown

## 基础信息（不变）
- 姓名：林远
- 年龄：16岁（开书时）
- 外貌：瘦高，左手有神秘纹身，常穿灰布衣
- 口头禅："有意思。"（遇到挑战时）
- 核心动机：找到失踪的师父

## 状态快照

### 卷一结束时
- 实力：炼气三层 → 筑基初期
- 性格变化：从胆怯内向 → 开始有主见，但仍不擅社交
- 关键经历：发现师父失踪真相、击败第一个强敌赵九
- 当前情感：对师父的愧疚，对修炼的渴望
- 人际关系：
  - 苏晴：从陌生 → 信任的战友
  - 赵九：从恐惧 → 战胜后释然
  - 陈老：从依赖 → 失去后独立

### 卷二当前（第23章后）
- 实力：筑基中期
- 性格变化：变得果断，但开始有些冷酷（过度补偿之前的软弱）
- 关键经历：加入宗门、在比武中受重伤
- 当前情感：对力量的渴望开始吞噬理性
- 人际关系：
  - 苏晴：产生暧昧但因性格变化出现裂痕
  - 新角色·李师兄：亦师亦友但有隐藏目的
```

**关键规则**：每卷结束时，为每个活跃角色生成状态快照。下一卷创作时加载最新快照而非原始设定。

## 九、context_assembler —— 分层记忆拼装器

这是整个系统的核心引擎。每章写作前，它负责：

```python
def assemble_context(novel_dir, current_chapter):
    """为当前章节组装写作上下文"""
    context = []
    
    # Layer 0: 基因层（永远加载）
    context.append(read_file(f"{novel_dir}/全书基因.md"))          # <2000字
    
    # Layer 1: 卷级记忆
    current_vol = get_current_volume(current_chapter)
    context.append(read_file(f"{novel_dir}/{current_vol}/卷概要.md"))  # <3000字
    
    # Layer 2: 弧线记忆
    current_arc = get_current_arc(current_chapter)
    context.append(read_file(f"{novel_dir}/{current_vol}/{current_arc}.md"))  # <2000字
    
    # Layer 3: 章级记忆
    summaries = get_recent_summaries(novel_dir, current_vol, n=3)   # 最近3章摘要
    context.append(summaries)
    prev_chapter_tail = get_chapter_tail(current_chapter - 1, chars=500)
    context.append(prev_chapter_tail)
    
    # 按需检索
    foreshadows = get_due_foreshadows(current_chapter)   # 本章需回收的伏笔
    if foreshadows:
        context.append(foreshadows)
    
    characters = get_chapter_characters(current_chapter)  # 本章出场角色最新状态
    context.append(characters)
    
    return "\n---\n".join(context)
    # 总计约 9000-12000字，远小于模型上下文窗口
```

**关键设计**：常驻记忆控制在 ~9000字，按需记忆只在需要时加载，确保几十万字的小说也不会撑爆上下文。

## 十一、质量控制体系

### 章级质量（每章写完后）
| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 字数 | 脚本统计 | 3000-5000字 |
| 术语一致 | 脚本比对术语表 | 0偏差 |
| AI词汇密度 | 黑名单词频 | <3次/章 |
